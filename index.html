The Challenege of Non-Static Media

When adding an element to a website, developers may consider the following:

  • Purpose of the element
  • User interaction with and view the element
  • Presence and complexity of the element

The majority of the elements used in web design would be considered "static" elements (ie. body text, headers, links, or even static images). Since HTML is static by nature, these elements are (almost) unanimously and unifromally supported accross web browsers.

Unfortunately, the same cannot be said for "non-static" elements. Video formats, and transparent video formats in particular, are very lacking in consistent browser support. Thus, when adding non-static we one must also consider the following:

  • File size and impact on performance
  • Use of transparency
  • Cross-browser support

Ultimately, these three (and some of the first three) help us determine what animated media format is best suited for our purposes.

Do not fear, all hope is not lost. For most readers, two heroic media formats will serve our purposes well.

GIF vs MP4

Without diving into their histories, the two most common animated fromats are GIF and MP4 (specifically using H.264 compression). It should be noted that MP4 is a video format, but GIF is still considered an image format. Both have very consistent cross-browser support, but which one is right for you?

File Size and MP4

While GIF files are very easy to implement, they have extremely poor compression compared to MP4. This makes GIFs perfectly acceptable when used lightly, but websites that use a LOT of animated content (i.e. twitter.com and giphy.com), file sizes need to be controlled.

MP4's on the otherhand have AMAZING compression. Looking below, we have a gif generated from a youtube video using gifrun.com that came out to ~3.5M. Using a media conversion tool like FFmpeg, we can convert the GIF into an MP4 that is only 373KB. The MP4 version is a mere 10% of the original GIF!

  • Format: GIF
  • File Size: 3.5MB
  • Format: MP4 (H.264)
  • File Size: 373KB
Video Credit:SmallButera

FFmpeg is completely free and becomes handly tool once you get the hang of it. This fantastic article by Billy Hoffman of Rigor Inc further breaks down the benefits and practices for using MP4s in place of GIFs on your website. He also offers this handy FFmpeg command that is optimized for web:

ffmpeg -i animated.gif -movflags faststart -pix_fmt yuv420p -vf "scale=trunc(iw/2)*2:trunc(ih/2)*2" video.mp4

Transparent Media !!!WIP!!!

So we know that MP4 with GIF as a fallback will cover most of our bases. But what if we REALLY wanted to use a transparent video?

<video muted autoplay playsinline loop>
    <!-- For Safari -->
    <source src="assets/DELM-Brian-02-1.mov" type='video/mp4; codecs="hvc1"' />

    <!-- For Firefox and Chrome -->
    <source src="assets/DELM-Brian-02-1.webm" type="video/webm" />

    <!-- Backup in case someone uses edge -->
    <img src="assets/Brian-02.gif" type="img" />
</video>
ffmpeg -i your_original_file.mov -c:v libvpx-vp9 -lossless 1 -auto-alt-ref 0 new_webm_file.webm

Popular combos:

  • webm - firefox, chrominium, edge (VP9)
  • mov - safari and safari mobile (HEVC)
  • GIF and PNG backups for other browsers
